## 🎨 Want to Use This Theme for Your Own Blog?

This website is built with Jekyll and uses the Chirpy theme with custom modifications. Here's how you can use it for yourself:

### Quick Start

1. **Fork this repository** or clone it:
   ```bash
   git clone https://github.com/gaurovgiri/gaurovgiri.github.io.git
   cd gaurovgiri.github.io
   ```

2. **Customize the configuration** in `_config.yml`:
   - Change the `title`, `tagline`, and `description`
   - Update `url` to your GitHub Pages URL
   - Replace social media links with your own
   - Update `email` and other personal information

3. **Run locally with Docker** (recommended):
   ```bash
   docker compose up
   ```
   Visit `http://localhost:4000` to see your site

4. **Or run with local Jekyll installation**:
   ```bash
   bundle install
   bundle exec jekyll serve
   ```

### Customizing the Theme

This project uses **local Chirpy theme files** instead of the gem-based theme, which means you can freely customize:

- **Layouts**: Edit files in `_layouts/` to change page structures
- **Includes**: Modify components in `_includes/` for reusable elements
- **Styles**: Customize SASS files in `_sass/` to change the look and feel
- **Assets**: Add your own images, fonts, and scripts in `assets/`

### Writing Posts

1. Create a new file in `_posts/` with the format: `YYYY-MM-DD-title.md`
2. Add front matter at the top:
   ```yaml
   ---
   title: Your Post Title
   date: YYYY-MM-DD HH:MM:SS +/-TTTT
   categories: [Category1, Category2]
   tags: [tag1, tag2]
   ---
   ```
3. Write your content in Markdown below the front matter

### Deployment

- **GitHub Pages**: Push to `main` or `gh-pages` branch, and GitHub will build it automatically
- **Custom Domain**: Update `CNAME` file with your domain name
- **Docker**: Use the included Dockerfile for containerized deployment

### Project Structure

```
.
├── _data/              # Data files (contacts, sharing options)
├── _includes/          # Reusable HTML components
├── _layouts/           # Page templates
├── _posts/             # Your blog posts
├── _sass/              # SASS stylesheets
├── _tabs/              # Static pages (About, Archives, etc.)
├── assets/             # Images, CSS, JS files
├── tools/              # Build and development scripts
├── _config.yml         # Main configuration file
└── Gemfile             # Ruby dependencies
```
